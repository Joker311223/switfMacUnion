MenuBarManager 菜单栏图标管理工具

macOS 菜单栏空间有限，当运行的应用过多时，部分图标会被系统截断无法显示。MenuBarManager 常驻菜单栏，自动扫描所有菜单栏图标，支持一键折叠与展开，让菜单栏始终整洁有序。

核心功能
整洁菜单栏，从此开始
不再为菜单栏显示不全而烦恼，所有图标尽在掌控。
🔍
自动扫描菜单栏应用
实时检测所有正在运行且具有菜单栏图标的应用，包括纯菜单栏 App（.accessory）和普通 App，应用启停自动同步刷新。
👁
一键隐藏 / 显示图标
在弹出面板或偏好设置的应用管理表格中，对每个图标单独切换显示状态，设置即时生效，下次启动自动恢复。
⋯
溢出展开面板
点击菜单栏中的「⋯」按钮，以浮动弹出面板的方式列出所有被隐藏的应用图标，不占用额外菜单栏空间。
🖱
快捷激活应用
在展开面板中点击任意应用名称，直接激活切换到该应用，无需打开 Dock 或使用 ⌘Tab 切换。
🎛
最大显示数量设置
在偏好设置中通过步进器调整菜单栏最多展示几个图标，超出数量的图标自动归入溢出面板。
🌙
纯菜单栏模式
可在偏好设置中隐藏 Dock 图标，以纯菜单栏 App 方式运行，不占任何屏幕空间，开机自启后静默常驻。
💾
本地持久化
每个图标的隐藏/显示状态保存在本地 JSON 文件中，重启后自动恢复所有设置，数据不上云。
🔄
动态实时更新
监听 NSWorkspace 应用启动/退出通知，图标列表自动刷新，新安装的菜单栏 App 启动后立即被检测到。
使用方式
三步解决菜单栏拥挤
从启动到整理完成，整个流程不超过一分钟。
1
启动 MenuBarManager
运行后自动进入菜单栏，不显示 Dock 图标（可在偏好设置中切换）。菜单栏右侧出现「⋯」图标标志启动成功。
2
打开应用管理
点击「⋯」→「偏好设置」→「应用管理」，或直接在弹出面板中操作。所有正在运行的菜单栏应用会自动列出。
3
按需隐藏图标
对不常用的应用点击「隐藏」，它们从菜单栏消失但仍在后台运行。需要时点击「⋯」展开即可访问。
4
设置最大数量
在「通用」偏好中设置菜单栏最多显示几个图标。超出数量的图标自动折叠，菜单栏始终保持整洁。
快捷键
键盘操作，快人一步
常用操作均有快捷键支持，无需每次鼠标点击状态栏图标。
快捷键功能描述
⌘, 打开偏好设置窗口
⌘R 刷新菜单栏应用列表
⌘Q 退出 MenuBarManager
偏好设置
两大设置面板，完全掌控
通用设置调整整体行为，应用管理精细控制每个图标。
⚙️ 通用
  • 菜单栏最大显示图标数量（步进器，范围 1~20）
  • 显示「⋯」溢出展开按钮（开关）
  • 登录时自动启动
  • 在 Dock 中隐藏图标（纯菜单栏模式）
📋 应用管理
  • 表格列出所有运行中的菜单栏应用
  • 显示应用图标、名称、类型（菜单栏 / 普通）
  • 逐条切换「隐藏 / 显示」状态
  • 一键刷新应用列表
构建与运行
一行命令，即刻运行
使用 Swift Package Manager 构建，无需 Xcode，无需额外依赖。
bash 
# 克隆仓库
git clone https://github.com/joker311223/switfMacUnion.git
cd switfMacUnion/MenuBarManager

# 构建（Debug）
swift build

# 运行（启动后将出现在菜单栏右侧）
.build/debug/MenuBarManager

# Release 版本
swift build -c release
.build/release/MenuBarManager

⚠️ 要求 macOS 13+ 及 Swift 5.9+（Xcode 15 或 Swift 命令行工具）
💡 首次运行时 macOS 可能需要授予「辅助功能」权限以读取菜单栏状态
